### 发现的 Bug 列表

#### BUG-1（严重）: main.py:16 多余的 input 语句

- **文件**: `src/main.py` 第 16 行
- **现象**: `input_student_info()` 函数中有两行 `input("请输入学号: ").strip()`，第二行（第16行）是多余的，会吞掉原本应赋给 `age` 的输入
- **影响**: 正常交互使用时，用户输入年龄的那一行数据被多余 input 吃掉，导致年龄和成绩字段错位或触发格式错误
- **复现步骤**:
  1. 运行 `python main.py`
  2. 选择 1（添加学生）
  3. 输入学号后，再输入姓名，然后程序多显示一次"请输入学号"
  4. 输入的数据流整体偏移
- **建议修复**: 删除 `main.py` 第 16 行的 `input("请输入学号: ").strip()`

#### BUG-2（中等）: 添加学生时输入格式错误后流程中断

- **文件**: `src/main.py` `input_student_info()` 函数
- **现象**: 当年龄或成绩输入非法值时，`try/except ValueError` 捕获异常后直接 `return None`，用户需要重新从菜单开始，已输入的学号和姓名全部丢失
- **影响**: 用户体验差，一个字段出错就要全部重来
- **建议修复**: 在输入年龄和成绩时加入循环重试逻辑，直到输入合法为止

#### BUG-3（低）: input 提示语标点不一致

- **文件**: `src/main.py`
- **现象**: 第14行 `input("请输入学号: ")` 使用英文冒号，第15行 `input("请输入姓名：")` 使用中文冒号
- **影响**: 视觉不统一，不影响功能
- **建议修复**: 统一为中文冒号或英文冒号

#### BUG-4（低）: 空学号和空姓名可以创建学生

- **文件**: `src/main.py` + `src/student.py`
- **现象**: 用户不输入学号或姓名直接回车，会被 `strip()` 处理为空字符串，但系统不会拦截，直接创建学号为空的学生
- **影响**: 可以创建无学号、无姓名的无效学生，后续查找/删除时可能产生混淆
- **建议修复**: 在 `input_student_info()` 或 `StudentManager.add_student()` 中加入空值校验

#### RISK-1（设计风险）: get_all_students() 返回内部列表引用

- **文件**: `src/student_manager.py` 第 41 行
- **现象**: `get_all_students()` 直接返回 `self.students`，外部代码可以绕过 `add_student`/`remove_student` 直接修改内部列表
- **影响**: 破坏封装性，外部可以直接 `append`、`remove` 等操作
- **建议修复**: 返回列表副本 `return self.students[:]` 或使用元组 `return tuple(self.students)`

#### RISK-2（设计风险）: 年龄/成绩无范围校验

- **现象**: Student 类不校验年龄和成绩的取值范围，可以创建年龄为负数、成绩为 999 的学生
- **影响**: 数据不合理但不会崩溃
- **建议修复**: 后续在 `input_student_info()` 或 `Student.__init__` 中加入范围校验
