# DESIGN

## 1. 项目目标
使用 Python 面向对象实现一个控制台学生管理系统。

## 2. 核心对象
### Student
- 表示单个学生
- 属性：学号、姓名、年龄、性别、成绩

### StudentManager
- 管理多个 Student 对象
- 负责添加、删除、修改、查找、显示学生信息

## 3. 程序入口
### main
- 显示菜单
- 接收用户输入
- 调用 StudentManager 的方法

## 4. 当前设计决策
- 使用列表保存学生对象
- 学号唯一
- 先做控制台版本
- 先不做文件持久化

## 5. 后续可扩展方向
- 保存到 JSON
- 排序功能
- 统计功能

## Student 类

### 作用
表示单个学生对象，负责保存一个学生的信息。

### 属性
- student_id：学号
- name：姓名
- age：年龄
- gender：性别
- score：成绩

### 第一版能力
- 创建学生对象时保存学生基本信息
- 通过 `__str__` 返回可读的学生信息字符串

### 当前说明
- 第一版暂不处理学号重复校验
- 第一版暂不做严格输入校验
- 学号重复检查由 `StudentManager` 负责

## StudentManager 类

### 作用
管理多个 Student 对象，负责学生信息的增删改查。

### 属性
- students：用于保存所有 Student 对象的列表

### 第一版方法
- `find_student_by_id(student_id)`
  - 根据学号查找学生
- `add_student(student)`
  - 添加学生对象，学号重复时返回失败
- `remove_student(student_id)`
  - 根据学号删除学生
- `update_student(student_id, name=None, age=None, gender=None, score=None)`
  - 根据学号修改学生信息，未传入的字段保持不变
- `get_all_students()`
  - 获取当前所有学生对象

### 当前设计说明
- 内部使用列表保存学生对象
- 学号唯一
- 删除和修改操作都基于学号查找
- 第一版返回布尔值表示操作是否成功

## main.py 菜单框架

### 作用
负责程序入口、菜单展示、用户输入和功能分发。

### 当前功能
- 显示菜单
- 添加学生
- 查看所有学生
- 查找学生
- 修改学生
- 删除学生
- 退出系统

### 设计说明
- `main.py` 只负责交互和调度，不直接处理学生管理的核心逻辑
- 具体的增删改查由 `StudentManager` 完成
- 用户输入后统一使用 `strip()` 去除首尾空白，减少输入格式问题
